## 1) Docelowe 4 akcje + mapping hotkeys (stałe)

W CodeScribe mamy na stałe 4 akcje (bez „5-tej opcji”):

1. **Hands-off (start/stop)** — toggle bez trzymania
2. **Hold-to-talk (RAW)** — dyktowanie, „puść żeby zakończyć”
3. **Talk to AI** — rozmowa z asystentem
4. **Selected text → AI** — wysyłka zaznaczenia do asystenta

Wybrany mapping (minimalne konflikty):

- **Hands-off (RAW start/stop):** `double Ctrl`
- **Hold-to-talk (RAW):** `Ctrl+Option` (hold)
- **Talk to AI:** `Ctrl+Option+Shift` (hold)
- **Selected text → AI:** `Ctrl+Option+Command` (hold)

Uwaga: przy tej bazie **`Ctrl+Shift`** i **`Ctrl+Cmd`** _same_ nic nie robią (Option jest częścią bazy), co jest świadomym wyborem: nie chcemy psuć skrótów typu `Ctrl+K`, `Ctrl+C/V` itp.

### Source of truth (config)

Konfiguracja hotkeys żyje w `~/.codescribe/.env`:

- `HOLD_MODS=ctrl_alt`
- `TOGGLE_TRIGGER=double_ctrl`
- `HOLD_EXCLUSIVE=0` (włącza warstwę trybów Shift/Cmd na bazie hold)

Dodany sanity-check: jeśli ktoś ustawi `TOGGLE_TRIGGER=double_ctrl` + `HOLD_MODS=ctrl`, config automatycznie przełączy hold na `ctrl_alt` (żeby nie skończyć ze „hold nie działa”).

## 2) Tray UX: „niech klikanie działa” (runtime + .env)

Problem: wcześniej tray potrafił tylko zapisać `.env`, a runtime hotkeys niekoniecznie się przełączały od razu → chaos „działa / nie działa”.

Zmiany:

- Tray teraz aplikuje zmiany **od razu** do runtime hotkeys (nie tylko zapis `.env`).
- W `Tools → Advanced… → Hotkeys` dodane:
  - **Hold key** jako radio: `Ctrl` / `Ctrl+Option` / `Ctrl+Shift` / `Ctrl+Command`
  - **Apply recommended preset (Ctrl+Option + Double Ctrl)** — ustawia stabilny zestaw pod 4 akcje
  - **Copy hotkeys cheatsheet** — kopiuje do schowka aktualny mapping

## 3) Quick Notes (voice → text notes)

Quick Notes to _ten sam pipeline nagrywania/transkrypcji_, ale z dodatkowym „persist do notatek”.

UI:

- `Tools → Notes → Quick Notes (save)` — włącza dopisywanie notatek
- `Tools → Notes → Save-only (no paste)` — jeśli ON, to **nie wkleja** do aktywnej aplikacji (tylko zapis)

Persist:

- domyślnie zapis do: `~/.codescribe/notes/YYYY-MM-DD.md`
- override: `CODESCRIBE_NOTES_DIR=/some/path`

Implementacja persist: `core/state/notes.rs`.

## 4) Crash fix: badge window use-after-free

Objaw: `EXC_BAD_ACCESS` / `objc_msgSend` / selector `setFrameOrigin:` na main thread.

Przyczyna: update pozycji badge window był planowany na main queue, ale okno mogło zostać zamknięte/zastąpione zanim blok się wykona → wskaźnik okna stawał się nieaktualny.

Fix: przed `setFrameOrigin:` sprawdzamy, czy pointer nadal jest „current” w stanie badge.

- plik: `app/ui/mod.rs` (commit `fa70ec0`)

---

## 5) macOS notifications: koniec z Script Editor

Objaw: po transkrypcji/notatce pojawiała się notyfikacja, a kliknięcie otwierało Script Editor.

Przyczyna: notyfikacje były robione przez `osascript -e 'display notification …'`, więc macOS przypisywał je do Script Editora/osascript.

Fix: wysyłamy notyfikacje z CodeScribe przez `NSUserNotification` (legacy, ale OK dla dev tool).

- nowy plik: `app/os/notifications.rs`
- podmiana użyć w: `app/controller/mod.rs`, `app/ui/tray/handlers.rs`, `bin/codescribe.rs`

## 6) Instrukcja „jak używać” (dla przyszłości)

1. Zainstaluj appkę (dev):

```sh
CODESCRIBE_CODESIGN_IDENTITY="CodeScribe Dev" make install-app
```

2. Ustaw hotkeys:
   `Tray → Tools → Advanced… → Hotkeys → Apply recommended preset (Ctrl+Option + Double Ctrl)`

3. 4 podstawowe akcje:

- `double Ctrl` — start/stop (hands-off)
- `Ctrl+Option` — RAW dictation (hold)
- `Ctrl+Option+Shift` — talk to AI (hold)
- `Ctrl+Option+Cmd` — selected text → AI (hold)

4. Quick Notes (opcjonalne):
   `Tray → Tools → Notes → Quick Notes (save)`
