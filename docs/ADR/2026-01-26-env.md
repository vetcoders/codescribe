# ADR 2026-01-26 — ENV Documentation Baseline (Archived)

Status: archived

## Context

This ADR originally contained a full, copy-pasted ENV contract snapshot.
Over time, that snapshot diverged from runtime reality (legacy flags, old defaults,
and duplicate sections), which created confusion.

## Decision

This file is now kept as a historical marker only.
It is **not** the active ENV contract.

The current, authoritative sources are:

- `docs/ENV_REGISTRY.toml` — canonical variable registry and metadata.
- `docs/env.md` — human-oriented runtime guide.
- `core/config/default_env.txt` — shipped app defaults template.

## Consequences

- We avoid maintaining multiple full ENV specs in different docs.
- Any future ENV changes should update `docs/ENV_REGISTRY.toml` first.
- `docs/env.md` should summarize behavior, but never redefine a second source of truth.

## Historical note

The previous content in this ADR included legacy settings from pre-event-pipeline
iterations (for example old buffered-stream assumptions and stale defaults).
Those details were intentionally removed to prevent accidental reuse.
